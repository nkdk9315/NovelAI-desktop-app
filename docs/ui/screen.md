## 2. S1: プロジェクト一覧画面

**種類**: 独立ページ（`/`）
**関連機能**: F1.1, F1.2, F1.3, F1.4

```
+------------------------------------------------------------------+
|                                                                    |
|                     NovelAI Desktop                                |
|                                                                    |
|  [+ 新規プロジェクト]                            [設定]           |
|                                                                    |
|  +------------------------------------------------------------+   |
|  | プロジェクト名A        単純生成    2026-04-01    [開く][削除] |   |
|  +------------------------------------------------------------+   |
|  | プロジェクト名B        単純生成    2026-03-28    [開く][削除] |   |
|  +------------------------------------------------------------+   |
|  | プロジェクト名C        単純生成    2026-03-15    [開く][削除] |   |
|  +------------------------------------------------------------+   |
|                                                                    |
|                    プロジェクトがありません                          |
|                    （空状態メッセージ）                              |
+------------------------------------------------------------------+
```

### 構成要素

| 要素 | 説明 |
|------|------|
| アプリタイトル | "NovelAI Desktop" |
| 新規プロジェクトボタン | クリックでS8（プロジェクト作成ダイアログ）を開く |
| 設定ボタン | クリックでS3（設定ダイアログ）を開く |
| プロジェクトリスト | Card形式。各カードに名前、タイプ、作成日、操作ボタン |
| 開くボタン | プロジェクトを開きS2へ遷移。F1.3 + F1.5（未保存画像クリーンアップ） |
| 削除ボタン | S9（削除確認ダイアログ）を表示後、F1.4実行 |
| 空状態 | プロジェクトが0件の場合に表示 |

### 3.1 ヘッダー

**関連機能**: F2.4, F10.1-F10.6, F11

| 要素 | 説明 | 関連機能 |
|------|------|----------|
| 戻るボタン | S1（プロジェクト一覧）へ遷移 | — |
| プロジェクト名 | 現在のプロジェクト名を表示 | F1.3 |

### 3.2 左パネル（プロンプトエリア）

**幅**: 320px（w-80）、縦スクロール可能
**関連機能**: F3, F4, F5, F6, F7, F8, F9

#### セクション1: メインプロンプト（F3.1, F3.4, F3.5, F3.6）

```
+--------------------------------------+
| Main Prompt                          |
| +----------------------------------+ |
| | ポジティブプロンプト               | |
| | （テキストエリア）                 | |
| | オートコンプリート対応             | |
| +----------------------------------+ |
| [v Negative]                         |
| +----------------------------------+ |
| | ネガティブプロンプト               | |
| | （デフォルト値初期設定）           | |
| +----------------------------------+ |
| [+ Group]                            |
+--------------------------------------+
```

| 要素 | 説明 |
|------|------|
| ポジティブテキストエリア | メインプロンプト入力。F7オートコンプリート付き |
| ネガティブテキストエリア | 折りたたみ可能。デフォルトネガティブプロンプトで初期化 |
| グループ追加ボタン | PromptGroupPicker を開く。選択したグループのタグを累積追加 |

#### セクション2: キャラクター追加ボタン（F6.1, F6.5）

```
+--------------------------------------+
| + Character                          |
| [男] [女] [その他] [カスタム...]     |
+--------------------------------------+
```

| 要素 | 説明 |
|------|------|
| ジャンルボタン | システムジャンル（男/女/その他）+ カスタムジャンル |
| 上限制御 | 6人到達時は全ボタン無効化 |
| 自動適用 | ボタン押下で当該ジャンルのデフォルトグループタグを自動適用（F6.2） |

#### セクション3: キャラクターセクション × 0〜6（F3.2, F3.3, F6.3, F6.4, F6.6）

```
+--------------------------------------+
| Character 1 (女)               [x]  |
| +----------------------------------+ |
| | ポジティブプロンプト               | |
| | オートコンプリート対応             | |
| +----------------------------------+ |
| [v Negative]                         |
| +----------------------------------+ |
| | ネガティブプロンプト               | |
| +----------------------------------+ |
| Position X: [====o======] 0.5        |
| Position Y: [====o======] 0.5        |
| [+ Group]                            |
+--------------------------------------+
```

| 要素 | 説明 |
|------|------|
| ヘッダー | キャラクター番号 + ジャンル名 + 削除ボタン |
| プロンプト | ポジティブ/ネガティブ（メインと同構成） |
| ポジションスライダー | center_x, center_y（0.0〜1.0）|
| グループ追加 | PromptGroupPicker を開く |

#### セクション4: アーティスト/スタイル（F9.4, F9.5）

```
+--------------------------------------+
```

| 管理ボタン | S6（スタイルプリセット管理モーダル）を開く |

#### セクション5: Vibe（F8.4, F8.5, F8.6）

### 3.3 中央パネル（画像表示エリア）

**幅**: 残りスペース（flex-1）
**関連機能**: F2.1, F2.2, F2.6, F1.6

```
+--------------------------------------+
|                                      |
|                                      |
|          Generated Image             |
|          (object-contain)            |
|                                      |
|          [Loading Spinner]           |
|          (生成中)                    |
|                                      |
|          [Empty State]               |
|          (未生成時)                   |
|                                      |
+--------------------------------------+
|  [Generate]  [Save] [Save All] [Del] |
+--------------------------------------+
```

| 要素 | 説明 |
|------|------|
| 画像表示 | 生成直後はbase64、履歴選択時はassetプロトコル経由 |
| ローディング | 生成中にスピナーオーバーレイ表示 |
| 空状態 | 未生成時のプレースホルダー |
| Generateボタン | 画像生成実行。生成中は無効化 |
| Saveボタン | 現在表示中の画像を保存（未保存画像のみ表示） |
| Save Allボタン | プロジェクト内の全画像を保存 |
| Deleteボタン | 現在表示中の画像を削除（確認ダイアログ付き） |

**幅**: 256px（w-64）
**関連機能**: F2.3, F1.6

```

| | img |  | img |       |
| | [*] |  | [ ] |       |
| +-----+  +-----+       |
| +-----+  +-----+       |
| | img |  | img |       |
| | [ ] |  | [*] |       |
| +-----+  +-----+       |
|        ...              |

```

| 要素 | 説明 |
|------|------|
| フィルタ切替 | All / Saved のみ |

```
| | > デフォルト女性外見  * |  | ジャンル: [女         v]     |
| |   ポーズ集            |  | 用途: [character     v]     |
| |   背景タグ            |  | デフォルト: [x]              |
```

## 12. レスポンシブ方針

- **ウィンドウサイズ**: 初期 1400×900、リサイズ可能
- **左パネル**: 固定幅 320px
- **右パネル**: 固定幅 256px
- **中央パネル**: 残りスペースをflex-1で占有
- **モバイル対応**: 不要（デスクトップ専用）
- **最小幅**: 左 + 右パネル + 最低限の中央パネル ≈ 800px 程度

- UI フォントを Inter → **IBM Plex Sans** + IBM Plex Sans JP + Noto Sans JP に移行
- モノスペースフォントを JetBrains Mono（Variable）+ IBM Plex Mono のフォールバックに統一
- Tailwind v4 `@theme` 構文を採用。カラートークンは `--color-*` が `--primary` などの CSS 変数を参照する二段構成に
- レスポンシブ性を向上（S1 カードグリッド、モーダル最大幅）
- 詳細: `docs/ui/DESIGN_SYSTEM.md` §3 Typography

### A2. PR #23 — Primary hue 195 → 230（`34162d1`, 2026-04-XX）

- `--primary` / `--ring` / `--sidebar-ring` の hue を **cyan(195) → blue(230)** に変更
- ダーク `oklch(0.72 0.10 230)` / ライト `oklch(0.55 0.11 230)`
- アクセントを「青シアン」から「ダークブルー」へ寄せて、全画面のプライマリカラー（Generate ボタン / フォーカスリング / アクティブタブ）に反映

### A3. PR #24 — カスタム画像サイズ入力（`1318294`, 2026-04-XX）

- S2 ヘッダーの `SizeSelector` にカスタム入力フィールドを追加（任意の幅/高さを数値入力）
- プリセットはグループ化して表示（`Portrait / Landscape / Square / Wallpaper` などのカテゴリ化）
- Tauri コマンドではなく純粋フロントエンド実装（`generation-params-store` の `width`/`height` を直接更新）

### A4. PR #25 — サイドバードラッグリサイズ（`90e1ef2`, 2026-04-XX）

- 左右サイドバー（S2 の aside）を**固定幅から可変幅**に変更
- 実装: `src/stores/layout-store.ts`（`leftSidebarWidth` / `rightSidebarWidth`）+ `ResizeHandle` コンポーネント
- 範囲: 左 240〜560 px（初期 320）、右 200〜480 px（初期 256）
- 永続化: Zustand `persist`、`localStorage` キー `layout-state`
- 「12. レスポンシブ方針」の「左/右パネル 固定幅」の記述はこの Addendum で上書きされる
