# ユーザーフロー定義書

## 1. 画面遷移図

```
[起動]
  │
  v
[S1: プロジェクト一覧] ──── [S3: 設定ダイアログ]
  │         │
  │    [S8: 作成ダイアログ]
  │         │
  │         v
  │    新規プロジェクト作成
  │         │
  v         v
[S2: 生成画面（3パネル）] ← プロジェクト選択
  │
  ├── [S3: 設定ダイアログ]
  ├── [S4: プロンプトグループ管理モーダル]
  ├── [S5: Vibe管理モーダル]
  ├── [S6: スタイルプリセット管理モーダル]
  └── [S7: 画像詳細オーバーレイ]
```

## 3. フロー詳細

### UF1: アプリ初回セットアップ

**前提条件**: 初回起動、APIキー未設定
**ゴール**: APIキーを設定してAnlas残高を確認

```
1. アプリ起動 → S1（プロジェクト一覧）表示
2. [設定] ボタンをクリック → S3（設定ダイアログ）表示
3. APIキーを入力 → [保存] クリック
4. システムがAPIクライアントを初期化
5. Anlas残高を自動取得・表示
6. 必要に応じてデフォルト生成パラメータを設定
7. [OK] でダイアログを閉じる → S1に戻る
```

**エラー時**: APIキーが無効な場合、エラーメッセージを表示し再入力を促す

---

### UF2: プロジェクト作成

**前提条件**: S1表示中
**ゴール**: 新しいプロジェクトを作成して生成画面へ遷移

```
1. [+ 新規プロジェクト] クリック → S8（作成ダイアログ）表示
2. プロジェクト名を入力
3. タイプを選択（現在は「単純生成」のみ）
4. [参照...] で保存先ディレクトリを選択（OSネイティブダイアログ）
5. [作成] クリック
6. システムがプロジェクトをDBに作成
7. S2（生成画面）へ自動遷移
```

**バリデーション**: 名前が空の場合は作成ボタン無効化

---

### UF3: プロジェクトを開く

**前提条件**: S1表示中、プロジェクトが1件以上存在
**ゴール**: 既存プロジェクトを開いて生成作業を再開

```
1. プロジェクト一覧から対象の [開く] をクリック
2. システムが未保存画像をクリーンアップ（F1.5）
   - is_saved=false の画像ファイルを削除
   - 対応するDBレコードを削除
3. S2（生成画面）へ遷移
4. 保存済み画像が右パネル（履歴）に表示
```

---

### UF4: プロジェクト削除

**前提条件**: S1表示中
**ゴール**: 不要なプロジェクトを削除

```
1. プロジェクト一覧から対象の [削除] をクリック
2. S9（削除確認ダイアログ）表示
   「プロジェクト「XXX」を削除しますか？この操作は元に戻せません。」
3. [削除] クリック
4. システムがDBレコードを削除
   （画像ファイルはユーザーディレクトリに残る / 合わせて削除するかは要検討）
5. プロジェクト一覧を更新
```

---

### UF5: 基本的な画像生成

**前提条件**: S2表示中、APIキー設定済み
**ゴール**: プロンプトを入力して画像を生成

```
1. 左パネルのメインプロンプトにテキストを入力
   - 入力中にオートコンプリート候補が表示（F7）
   - Tab/Enter で候補を選択
2. ネガティブプロンプトは折りたたみを展開して編集（デフォルト値あり）
3. ヘッダーでコスト予測を確認（リアルタイム更新）
4. [Generate] クリック
5. ローディングスピナーが中央パネルに表示
6. 生成完了 → 画像が中央パネルに表示
7. 画像が右パネルの履歴に追加（未保存状態）
8. Anlas残高が更新
```

**エラー時**:
- Anlas不足 → ヘッダーに警告、Generateボタン無効化
- API エラー → エラーメッセージ表示

---

### UF6: キャラクター付き画像生成

**前提条件**: S2表示中
**ゴール**: 複数キャラクターを指定して画像を生成

```
1. UF5のステップ1-2（メインプロンプト入力）
2. キャラクター追加ボタンのジャンルを選択（例: [女]）
3. キャラクターセクションが追加される
   - デフォルトプロンプトグループのタグが自動適用（F6.2）
4. キャラクタープロンプトを入力/編集
5. ポジションスライダーで画像内位置を調整（center_x, center_y）
6. 必要に応じて追加キャラクターを追加（最大6人）
7. [Generate] クリック → 画像生成
```

**操作**:
- キャラクター削除: セクションの [x] ボタン
- グループ追加: [+ Group] でPromptGroupPickerを開く
- 順序: 追加順 = API送信順（use_order: true）

---

### UF7: プロンプトグループを使った生成

**前提条件**: S2表示中、プロンプトグループが登録済み
**ゴール**: 定義済みのタグセットをプロンプトに追加

```
1. メインプロンプトまたはキャラクターセクションの [+ Group] クリック
2. PromptGroupPicker が表示
   - ジャンルフィルタ、使用タイプフィルタ、名前検索が利用可能
3. グループを選択
4. グループのタグがプロンプトに累積的に追加
5. 必要に応じて複数グループを追加
6. プロンプトを確認・編集 → 生成
```

---

### UF8: 画像の保存と管理

**前提条件**: S2表示中、画像が生成済み
**ゴール**: 気に入った画像を保存、不要な画像を削除

```
■ 個別保存
1. 中央パネルに画像を表示（生成直後 or 履歴からクリック）
2. [Save] クリック → 画像が保存状態に変更
3. 右パネルの保存インジケーターが更新

■ 全保存
1. [Save All] クリック
2. プロジェクト内の全未保存画像が保存状態に変更

■ 削除
1. 中央パネルに画像を表示
2. [Delete] クリック → 確認ダイアログ
3. 確認後、ファイルとDBレコードを削除
4. 履歴から除去、次の画像を表示

■ 履歴フィルタ
1. 右パネルの [All] / [Saved] 切替
2. [Saved] で保存済み画像のみ表示
```

---

### UF9: 画像詳細の確認

**前提条件**: S2表示中、画像が生成済み
**ゴール**: 画像のパラメータやプロンプトを確認

```
1. 右パネルの履歴サムネイルをダブルクリック
   （またはコンテキストメニュー等で「詳細」を選択）
2. S7（画像詳細オーバーレイ）が中央パネル上に表示
3. メタデータを確認:
   - Seed, Model, Size, Steps, Scale, Sampler
   - プロンプトスナップショット（JSON）
4. 必要に応じて [Save] / [Delete]
5. [x] で閉じる → 通常の中央パネルに戻る
```

---

### UF10: プロンプトグループの作成・管理

**前提条件**: S2表示中
**ゴール**: 再利用可能なタグセットを作成・編集

```
■ モーダルを開く
1. 左パネルのPromptGroupPicker内 [管理] ボタン等からS4を開く

■ グループ作成
1. S4で [+ 新規] をクリック
2. グループ名を入力
3. ジャンルを選択（男/女/その他/カスタム）
4. 使用タイプを選択（main/character/both）
5. タグを追加（オートコンプリート付き入力）
6. [保存] クリック

■ グループ編集
1. 一覧からグループを選択 → エディタに表示
2. 名前、ジャンル、用途、タグを編集
3. タグの並び替え（ドラッグまたは上下ボタン）
4. [保存] クリック

■ デフォルト設定
1. エディタで「デフォルト」チェックを有効化
2. 同ジャンルの既存デフォルトが自動解除
3. 以降、キャラクター追加時に自動適用

■ グループ削除
1. エディタで [削除] クリック → 確認
2. システムグループ（is_system=true）は削除ボタン非表示
```

---

### UF11: ジャンルの管理

**前提条件**: S4（プロンプトグループ管理モーダル）表示中
**ゴール**: カスタムジャンルを追加/削除

```
■ カスタムジャンル追加
1. S4のジャンルセクションで [+ 追加] クリック
2. ジャンル名を入力（例: 「宇宙人」「ロボット」）
3. Enter or [追加] で確定
4. 新ジャンルがタブ/ボタンとして追加
5. キャラクター追加ボタンにも反映

■ カスタムジャンル削除
1. カスタムジャンルを選択
2. [削除] クリック → 確認ダイアログ
   「ジャンルに紐づくプロンプトグループも削除されます」
3. 確認後、ジャンルと関連グループを削除

※ システムジャンル（男/女/その他）は削除不可
```

---

### UF12: Vibeのインポートと利用

**前提条件**: S2表示中
**ゴール**: Vibeをインポートして画像生成に適用

```
■ インポート
1. 左パネルのVibeセクション [管理] → S5表示
2. [インポート] クリック → ファイルピッカー
3. .naiv4vibe ファイルを選択
4. ファイルがアプリデータディレクトリにコピー、DBに登録
5. Vibe一覧に追加

■ 利用
1. S5を閉じる → 左パネルのVibeセクション
2. 使いたいVibeのトグルをON
3. strength スライダーを調整（0.0-1.0、デフォルト0.7）
4. information_extracted スライダーを調整（0.0-1.0、デフォルト0.7）
5. 選択カウント確認（最大10個）
6. [Generate] で生成 → Vibeのスタイルが反映

■ 削除
1. S5のVibe一覧で [Del] クリック → 確認
2. ファイルとDBレコードを削除
```

---

### UF13: Vibeエンコード

**前提条件**: S5表示中、APIキー設定済み、Anlas ≥ 2
**ゴール**: 画像からVibeを生成

```
1. S5の「画像からエンコード」セクション
2. [画像を選択...] → ファイルピッカーで画像を選択
3. モデルを選択（V4.5 Full 等）
4. [エンコード実行] クリック
5. 確認: 「2 Anlas を消費します。実行しますか？」
6. API呼び出し → .naiv4vibe ファイル生成
7. Vibe一覧に追加、Anlas残高更新
```

---

### UF14: スタイルプリセットの活用

**前提条件**: S2表示中
**ゴール**: アーティストタグとVibeの組み合わせを保存・適用

```
■ プリセット作成
1. 左パネルのアーティスト/スタイルセクション [管理] → S6表示
2. [+ 新規作成] クリック
3. プリセット名を入力
4. アーティストタグを追加（オートコンプリート付き）
5. Vibeを選択（チェックボックスで複数選択可）
6. [保存] クリック

■ プリセット適用
1. 左パネルのスタイルプリセットドロップダウンで選択
2. アーティストタグがメインプロンプトに追加
3. 対応するVibeが自動的にON状態に設定
4. 生成パラメータ → 生成
```

---

### UF15: パラメータ調整と生成

**前提条件**: S2表示中
**ゴール**: 生成パラメータを最適化して画像を生成

```
1. ヘッダーのパラメータコントロールを調整:
   - Model: ドロップダウンで選択
   - Size: Width × Height 入力（ピクセル制約内）
   - Count: 生成枚数
   - Sampler: ドロップダウンで選択
   - Steps: 数値入力（1-50）
   - Scale: 数値入力（0.0-10.0）
2. 変更のたびにコスト予測がリアルタイム更新（デバウンス付き）
3. Anlas不足の場合は警告表示
4. Opus無料生成条件を満たす場合は「Free」表示:
   - Vibe未使用、キャラクターリファレンス未使用
   - pixels ≤ 1,048,576、steps ≤ 28、tier = Opus
5. [Generate] で生成実行
```

## 4. 典型的なワークフロー（End-to-End）

### ワークフロー A: 初回起動から最初の画像生成まで

```
UF1 → UF2 → UF5
 (設定)  (プロジェクト作成)  (画像生成)
```

### ワークフロー B: 本格的なキャラクターイラスト制作

```
UF3 → UF10 → UF12 → UF6 → UF14 → UF8
 (開く) (グループ準備) (Vibe準備) (キャラ生成) (スタイル適用) (保存)
```

### ワークフロー C: プロンプトの試行錯誤

```
UF3 → UF5 → UF15 → UF5 → UF9 → UF8
 (開く) (生成) (パラメータ調整) (再生成) (詳細確認) (保存)
```

## 5. 状態遷移

### 画像の状態

```
[生成中] → [表示（未保存）] → [保存済み]
                  │
                  └→ [削除]
```

- 生成直後: is_saved=false、中央パネルに表示、履歴に追加
- 保存操作: is_saved=true に更新
- プロジェクト再オープン時: is_saved=false の画像は自動削除

### APIクライアントの状態

```
[未初期化] → [初期化済み] → [エラー]
                  │              │
                  └──────────────┘
                   (再初期化)
```

- 未初期化: 生成系の機能は全て無効
- 初期化済み: 全機能利用可能
- エラー: APIキー無効等。設定から再入力
